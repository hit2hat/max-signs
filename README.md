# Проверка подписи в мини-приложениях
В данном репозитории находятся примеры на 4 языках (js/ts, python, java и go) c примерами валидации подписи WebAppData.
Однако, важно обратить внимание: вы можете сделать данную проверку и на своем языке, если реализуете абстрактную логику.

## Абстрактная логика проверки
1. Вводные данные: `BOT_TOKEN` — токен бота, `USER_URL` — url, по которому открыто мини-приложение
2. Нужно получить fragment из USER_URL (данные после `#`). В нем в формате `key=value` есть параметры платформы, среди которых нужный нам — `WebAppData`.
3. Преобразуем значение WebAppData из `key=value` в `[['key', 'value']]`
4. Запоминаем оригинальный хеш (ключ равен `hash`) и исключаем его из этого массива
5. Далее применяем url-декодирование для всех значений (`value`), если ваша платформа этого не сделала сама.
6. Сортируем массив по ключам в алфавитном порядке `a -> z`
7. Формируем строку: `key1=value1\nkey2=value2`. Назовем её `launch_params`.
8. Вычисляем secret_key — ключ для проверки подписи параметров: для этого подписываем токен бота с помощью HMAC-SHA256 используя в качестве ключа строку `WebAppData`: `HMAC-SHA256(bot_token, 'WebAppData')`
9. Вычисляем собственную подпись: для этого подписываем полученную строку (шаг 7) с помощью HMAC-SHA256 используя secret_key: `HMAC-SHA256(launch_params, secret_key)`
10. Преобразуем подпись из шага 9 в hex-строку
11. Если hex-строка подписи равна значению параметра hash из оригинальной строки, то данные не подменили и все хорошо.
